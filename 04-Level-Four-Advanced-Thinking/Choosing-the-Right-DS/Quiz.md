

# Check Understanding – Choosing the Right Data Structure

## Instructions
Choose the **best answer** for each question.

---

### 1. Which data structure is most suitable when you need fast access by index?
A. Linked List  
B. Array  
C. Stack  
D. Tree  

---

### 2. Which data structure is best for implementing a First-In-First-Out (FIFO) system?
A. Stack  
B. Queue  
C. Tree  
D. Graph  

---

### 3. When frequent insertions and deletions are required, which data structure is generally preferred?
A. Array  
B. Linked List  
C. Heap  
D. Hash Table  

---

### 4. Which data structure is most appropriate for representing hierarchical relationships?
A. Array  
B. Queue  
C. Tree  
D. Hash Table  

---

### 5. Which data structure provides the fastest average-case search time?
A. Binary Search Tree  
B. Array  
C. Hash Table  
D. Linked List  

---

### 6. If memory usage must be minimized and data size is fixed, which should you choose?
A. Dynamic Array  
B. Linked List  
C. Static Array  
D. Graph  

---

### 7. Which data structure is best suited for undo/redo operations?
A. Queue  
B. Stack  
C. Tree  
D. Hash Map  

---

### 8. When you need to model relationships like social networks, which data structure is ideal?
A. Tree  
B. Graph  
C. Array  
D. Stack  

---

### 9. Which data structure maintains elements in sorted order automatically?
A. Array  
B. Hash Table  
C. Binary Search Tree  
D. Queue  

---

### 10. If you need both fast search and ordered traversal, which data structure is the best choice?
A. Hash Table  
B. Stack

## Learning Goal
These questions focus on **trade-offs** such as:
- Time vs space complexity
- Access speed vs flexibility
- Order preservation vs fast lookup
- Static vs dynamic memory usage
