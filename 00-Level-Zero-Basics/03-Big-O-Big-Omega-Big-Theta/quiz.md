## Questions

**Q1.** What does Big-O notation describe?  
A) Best-case complexity  
B) Average-case complexity  
C) Worst-case complexity  
D) Exact running time  

---

**Q2.** Big-Ω (Omega) represents:  
A) Upper bound  
B) Lower bound  
C) Tight bound  
D) Random bound  

---

**Q3.** Big-Θ (Theta) means:  
A) Upper bound only  
B) Lower bound only  
C) Both upper and lower bound  
D) Space complexity only  

---

**Q4.** If an algorithm runs in Θ(n), then it is also:  
A) O(n) only  
B) Ω(n) only  
C) Both O(n) and Ω(n)  
D) Neither  

---

**Q5.** Which notation is used to express the worst-case time complexity?  
A) Big-O  
B) Big-Ω  
C) Big-Θ  
D) None  

---

**Q6.** If an algorithm takes at least n steps, its complexity is:  
A) O(n)  
B) Ω(n)  
C) Θ(n²)  
D) O(1)  

---

**Q7.** Which of the following is a tight bound?  
A) O(n)  
B) Ω(n)  
C) Θ(n)  
D) O(n²)  

---

**Q8.** If T(n) = 3n + 5, its Big-O is:

B) O(n)

D) O(log n)

**Q9.** If T(n) = n² + n, its Big-Θ is:  
A) Θ(n)

**Q10.** Big-O ignores:

**Q13.** Can an algorithm be O(n²) and Ω(n) at the same time?
